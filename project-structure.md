# Project Structure

This project uses Godot 4.6 Standard with GDScript, Blender source assets, exported runtime assets, and simple automation scripts. The structure should stay predictable so humans and agents can find files quickly.

## Root Layout

```text
agentic-platformer/
  AGENTS.md
  README.md
  project-structure.md
  tools-comparison.md
  game/
  assets_src/
  assets_export/
  levels/
  tools/
  builds/
```

## Root Files

- `AGENTS.md`: mandatory instructions for Codex/opencode and other local agents.
- `README.md`: short human-facing project summary and run instructions.
- `project-structure.md`: this file; describes where project files should live.
- `tools-comparison.md`: engine/tool decision notes.
- `.gitignore`: ignores Godot cache, build output, temporary files, and local credentials.

## Godot Project

Godot files live under `game/`.

```text
game/
  project.godot
  scenes/
    main.tscn
    menus/
    levels/
    characters/
    props/
  scripts/
    main.gd
    player_controller.gd
    characters/
    gameplay/
    ui/
    tools/
  resources/
    characters/
    gameplay/
    materials/
  addons/
```

Guidelines:

- `game/project.godot` is the Godot project file.
- `game/scenes/` contains `.tscn` scene files.
- `game/scripts/` contains `.gd` scripts.
- Keep small scripts near their domain folder once the project grows.
- Commit `.gd.uid` files generated by Godot for scripts.
- Do not edit or commit `game/.godot/`; it is local cache/editor state.
- Prefer text `.tscn` scenes and `.tres` resources.
- Avoid large, hard-to-review monolithic scenes when several smaller scenes would be clearer.

## Scene Organization

Recommended Godot scene folders:

```text
game/scenes/
  main.tscn
  menus/
    main_menu.tscn
    pause_menu.tscn
  levels/
    level_001.tscn
  characters/
    player.tscn
    enemies/
  props/
    doors/
    platforms/
```

Guidelines:

- Use `main.tscn` as the entry scene.
- Put reusable character, prop, UI, and level scenes in domain folders.
- Keep generated level scenes separate from hand-authored reusable scenes.
- Prefer scene composition over copying nodes between scenes.

## Script Organization

Recommended script folders:

```text
game/scripts/
  main.gd
  characters/
    player_controller.gd
  gameplay/
    checkpoint.gd
    hazard.gd
  ui/
    main_menu.gd
    confirmation_dialog.gd
  tools/
    level_loader.gd
```

Guidelines:

- Use GDScript.
- Keep gameplay code readable and explicit.
- Prefer one main responsibility per script.
- Use exported variables for tunable movement, camera, and gameplay values.
- Do not introduce C#/.NET unless explicitly requested.

## Godot Runtime Assets

Runtime-ready assets imported by Godot should live under `game/assets/` only after we need Godot to manage them directly.

```text
game/assets/
  characters/
  props/
  environment/
  materials/
  audio/
  ui/
```

Guidelines:

- Use this folder for assets the game loads directly.
- Prefer `.glb`/`.gltf` for 3D models exported from Blender.
- Keep generated Godot `.import` files with their assets if Godot creates them.
- Do not put Blender source files here.

Current early-stage note: runtime exports currently live in root-level `assets_export/`. We can later copy or move approved runtime assets into `game/assets/` when they are ready to be imported by Godot.

## Blender Source Assets

Source art files live under `assets_src/`.

```text
assets_src/
  characters/
    basic_human.blend
  props/
  environment/
  materials/
  references/
```

Guidelines:

- `.blend` files are source files.
- Keep one asset or tightly related asset group per `.blend` file.
- Use clear names such as `basic_human.blend`, `factory_door.blend`, or `room_blockout.blend`.
- Do not commit Blender backup files such as `.blend1` or `.blend2`.
- Large `.blend` files should be handled with Git LFS.

## Exported Assets

Generated runtime assets live under `assets_export/`.

```text
assets_export/
  characters/
    basic_human.glb
    basic_human_preview.png
  props/
  environment/
```

Guidelines:

- `.glb`/`.gltf` files are exported from Blender and intended for Godot.
- Preview images are useful for quick review in Codex.
- Exports should be reproducible from source files and scripts where practical.
- Avoid hand-editing exported assets.

## Level Data

Source level data should live under `levels/`.

```text
levels/
  chapter_01/
    level_001.json
    level_001_notes.md
```

Guidelines:

- Keep gameplay-critical level structure in text data when practical.
- Use JSON for platforms, hazards, checkpoints, camera zones, and spawn points.
- Generate or update Godot scenes from level data rather than hiding all layout decisions inside editor-only scene files.
- Add validators before creating many levels.

## Tools And Automation

Automation scripts live under `tools/`.

```text
tools/
  blender/
    create_basic_human.py
  godot/
  validation/
  export/
```

Guidelines:

- Blender should be callable as `blender` from `PATH`.
- Use `blender --background --python tools/blender/script_name.py` for asset generation.
- Godot should be callable as `godot` from `PATH` when possible.
- Prefer repeatable scripts over manual export steps.
- Keep scripts small and documented by clear names.

## Build Output

Build output should live under `builds/`.

```text
builds/
  windows/
  android/
```

Guidelines:

- Do not commit build output.
- Export presets should be committed when they exist and do not contain local secrets.
- Local credentials, keystores, and export credentials must not be committed.

## Version Control Rules

- Commit source code, text scenes, resources, docs, scripts, level data, and intentional assets.
- Do not commit `game/.godot/`.
- Do not commit `builds/`.
- Do not commit Blender backup files.
- Use Git LFS for large binary assets such as `.blend`, `.glb`, `.png`, `.wav`, `.ogg`, and video files.
- Keep binary changes understandable through nearby notes, generator scripts, or clear commit messages.

## Recommended Growth Path

1. Keep the current simple prototype working.
2. Move the player cube into its own `game/scenes/characters/player.tscn`.
3. Import `assets_export/characters/basic_human.glb` into Godot.
4. Replace the cube visual with the humanoid model.
5. Add `game/assets/characters/` once Godot starts importing runtime assets.
6. Add `levels/` and validators before producing many levels.
